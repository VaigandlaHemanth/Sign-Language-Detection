# Sign Language Digit Detection

Recognize hand-signed digits (0–9) from images using two complementary approaches: a **MediaPipe landmark + classical ML** pipeline and a **CNN** trained on raw images. The landmark pipeline is fast, interpretable, and runs on CPU; the CNN is an image-based baseline for comparison.

---

## Approach

Instead of feeding raw pixels to every model, the primary pipeline extracts **21 hand landmarks (× x,y = 42 features)** per image with **MediaPipe Hands**, then trains classical classifiers on those features. This makes the models tiny, quick to train, and robust to lighting/background — the landmarks abstract away everything except hand geometry.

```
Image ─► MediaPipe Hands (21 landmarks) ─► 42 features ─► StandardScaler
      └─► (CNN track: 32×32 RGB ─► conv net)
                                              │
                                              ▼
                    Logistic Regression · Random Forest · SVM · CNN
```

Horizontal-flip augmentation effectively doubles the dataset. Classical models use an 80/20 stratified split with 5-fold cross-validation.

> **Honest note on the metrics.** The dataset is small (~330 images across 10 classes), so the ~96% test accuracy sits well above the ~76% cross-validation mean — a classic small-data overfitting gap. The cross-validation figure is the more trustworthy estimate of real-world performance; collecting more data per class is the clear next step. Evaluation uses classification metrics only (accuracy, per-class precision/recall/F1, confusion matrix) — earlier versions mistakenly reported MSE/R², which are regression metrics and do not apply to classification; those have been removed.

## Running it

```bash
pip install -r requirements.txt
```

1. **Collect data** — capture ~50 images per digit into `dataset/0` … `dataset/9` (webcam works fine), or point the notebook at a public sign-language-digit dataset.
2. **Train a landmark model** — run `svmfin.ipynb` (or `randomfin`/`logicfin`). It extracts landmarks, trains, and prints accuracy, cross-validation, a classification report, and a confusion matrix. Trained artifacts are saved locally (gitignored).
3. **CNN baseline** — run `american.ipynb` on 32×32 RGB images.

> The dataset and serialized models are **not** committed (see `.gitignore`) — they are regenerated by running the notebooks.

### Next step: real-time inference

The landmark pipeline is designed for live webcam use (MediaPipe streams landmarks in real time). A small OpenCV + saved-SVM loop would turn this into a true real-time demo — that script is the highest-impact addition and is on the roadmap.
